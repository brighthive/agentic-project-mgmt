# Jira Sprint Tracking

## Structure

```
jira/
├── TICKET_TEMPLATE.md  # Canonical Jira ticket format
├── CLAUDE.md           # This file — sprint data format spec
├── sprint/
│   ├── SPRINTS.md      # Master velocity table (all sprints)
│   ├── 1/ ... 7/       # Per-sprint directories
│   │   ├── stats.json
│   │   ├── tickets.json
│   │   ├── SUMMARY.md
│   │   ├── RELEASE_NOTES.md
│   │   ├── MARKETING_RELEASE_NOTES.md
│   │   ├── VALIDATION_REPORT.md
│   │   ├── SLACK_POST.md
│   │   └── PLAN.md (optional — pre-sprint planning)
```

## Required Files Per Sprint

Every sprint **MUST** have all 7 files. Generated by `/sprint-release` skill.

| File | Source | Content |
|------|--------|---------|
| `stats.json` | Jira API | Dates, ticket counts, points, completion %, WIP analysis, carryovers, PR stats |
| `tickets.json` | Jira API | Array of `{key, summary, status, assignee, points, type, epic, resolved}` |
| `SUMMARY.md` | LLM analysis | ASCII stats, goals assessment, PR-ticket linkage, WIP, team breakdown, problems, recommendations |
| `RELEASE_NOTES.md` | Jira + GitHub | Summary metrics, completed tickets w/ Jira links, PRs by repo w/ GitHub links |
| `MARKETING_RELEASE_NOTES.md` | Curated | Thematic grouping of completed work for GTM/sales audience |
| `VALIDATION_REPORT.md` | GitHub + Jira | PR-ticket linkage report, orphan PRs, unmatched tickets, estimation gaps |
| `SLACK_POST.md` | Curated | Reference copy of Block Kit message posted to `#releases` |

**Optional**: `PLAN.md` — pre-sprint planning with goals, planned tickets, dependencies.

## Sprint Close Process

Automated by `/sprint-release` skill. 6 phases:

1. **DISCOVER** — Auto-detect sprint from Jira board 152 (name, emoji, dates, ID)
2. **COLLECT** — Jira tickets (all statuses) + GitHub PRs (by date range, NOT dev-status)
3. **ANALYZE** — Generate SUMMARY.md with stats, WIP, linkage, team, recommendations
4. **RELEASE NOTES** — RELEASE_NOTES.md, MARKETING_RELEASE_NOTES.md, VALIDATION_REPORT.md
5. **PUBLISH** — Post to Slack `#releases`, create Notion Sprint page, update Active Sprint
6. **COMMIT** — Write all files to `jira/sprint/{N}/`, update SPRINTS.md, push

## PR Fetching

PRs are fetched by **date range from GitHub**, not from Jira dev-status (which is unreliable — returns 0 PRs when branches don't include `BH-XXX`).

Repos scanned: `brightbot`, `brighthive-webapp`, `brighthive-platform-core`, `brightbot-slack-server`, `brighthive-admin`, `brighthive-data-organization-cdk`

PR-ticket matching: branch name → PR title → PR body → Jira dev-status (fallback).
